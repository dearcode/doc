基本概念

用户信息
用户名，用户ID，密码，头像

用户名：
字符串，直接用手机号或者邮箱号或者啥能都行，不是乱码就行

用户ID：
数值，服务器根据在用户注册成功后生成的用户唯一ID，服务端为int64, java里应该是long的

密码：
字符串，随意

头像：
注册成功后，上传上来

SessionID：
用户登录后服务端会返回一个sessionID，登录成功后发消息啊，改资料啊，都要传这个ID

流程：
除用户注册外，所的操作都要带sessionID，服务器会对sessionID和IP地址校验，不一致的话会要求登录

用户注册
上传用户名和密码，成功会返回用户的ID

登录：
上传用户名和密码，成功后返回用户ID和sessionID

修改资料：
目前就有个头像，在登录后，上传头像和sessionID进行修改，成功返回0，失败错误码
